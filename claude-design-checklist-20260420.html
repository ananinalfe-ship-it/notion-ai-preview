Claude Design 实战
Claude Design 不是设计师：
它把「导演标准」变成
可执行的检查清单
以惊喜之夜官网为例
你以为 Claude Design 的价值是"帮你把页面画出来"。我这两天的体感恰好相反：它最厉害的地方，是逼你把「导演的标准」说清楚——并且把这些标准变成一张可以验收的清单。
01
先定「可演示版本」的定义
否则永远在做半成品

很多人做网站失败，不是不会做页面，而是不知道"什么叫可以给别人看"。

可演示版 · 检查清单
✓走完整个观众路径：首页 → 作品 → 详情 → 核心信息/视频 → 能返回继续浏览
✓每个关键页至少有：清晰标题 + 1个主视觉 + 1个信息区块 + 1个行动引导（CTA）
✓不做"未来会写"的空块：宁愿删掉，也不要留占位

Claude Design 在这里的作用：它会不断追问"你的用户下一步要去哪？"——这逼你把网站当成"动线"，而不是"图册"。

观众路径
02
观众路径优先
页面数量不重要，转场顺不顺才重要

我做官网时最大的判断标准，是用"导演语言"来描述每一页：

导演视角 · 页面诊断
✓首页不是介绍自己，是让观众"选入口"
✓作品页不是陈列，是给观众一个"浏览模式"
✓详情页不是堆信息，而是一个节奏：主视觉 → 关键一句话 → 证据 → 余味
可复用模块
03
统一「可复用模块」
比单页优化更省命

搭建中强行做了几件"全局统一"的事：视频双源切换、Logo 替换联动全站、剧照交互统一。

模块化 · 两个判断题
?这个功能会出现在多少页？超过 3 页，就必须模块化。
?这个模块的输入输出是什么？（比如：视频 ID、剧照数组、标题）
克制
04
「克制」不是少做
而是删掉不服务目标的部分

搭建过程中有个反复做的动作：删。

任何一块内容，如果不能让观众更快理解"我们是谁 / 我们能做什么 / 我们做过什么"，就删掉。

克制 · 灵魂拷问
?这块内容服务的是品牌定位，还是服务的是创作者的虚荣？
交互彩蛋
05
交互彩蛋要「收口」
可玩，但必须可解释

首页做了"三把钥匙"解锁隐藏卡的彩蛋。彩蛋很容易失控——做着做着变成独立小游戏。

彩蛋 · 收口规则
✓彩蛋只做一次机制，不做多套
✓彩蛋解锁的内容必须回到主叙事（不是无关惊喜）
✓彩蛋要有明确的结束（否则观众不知道自己"完成了什么"）
优先级
06
最后才做「锦上添花」层
SEO / OG / 性能

把优先级写在纸面上非常重要。我把待办分成三层：

待办优先级矩阵
高高优：contact 页升级、新 logo 替换、内容二次校对
中中：移动端真机测试、筛选器细节、i18n
低低：SEO/OG、WebP/懒加载、favicon、统一弹窗动画
每件事做之前问
?这件事不做，会影响"可演示"吗？
?这件事不做，会影响"可信度"吗？
AI 时代，最稀缺的
不是手速，是「裁决标准」

Claude Design 会让做网站变快，但真正决定你作品质感的，是你能否给出清晰的裁决：什么算对，什么算不对。

当你把这些标准写成清单，你就拥有了一个可复制的工作流：换一个项目、换一套素材，网站依然能稳定落地。
